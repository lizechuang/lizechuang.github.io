基于plist注册的WebView跳转路由

前言

前阵子在写项目中有一个需求，显示一些Html界面，后台返回html数据由我们这边显示，我们是直接用WKWebview去显示这个界面，并把包在tableViewCell中去显示。html数据中有些标签是有点击跳转到显示标签具体数据页面的业务，（标签后面带有具体数据请求URL的absoluteString）

这个时候可以有暴力的方法当点击的时候直接获取到URL的absoluteString数据然后直接用当前的navigationC去push到具体的页面VC去显示，这样确实Ojbk，但是在cell的回调方法中需要去解析这个absoluteString，然后再去判断要跳转到哪个页面。而且，假如我们当前的navigationC在第三个tab中，跳转的页面要去第二个tab，此时要怎么做，这个时候好像可以去获取tabbarVC再去调整，这个时候再回头去看当前VC的代理方法中，你会发现我怎么在这个代理方法中写了这么多判断，把tabbarVC和隔壁的navigationC耦合到我们这个VC，这种做法是不可取的。

还有一个很傻逼的地方，如果我们在需要用到WKWebView点击标签跳转的地方都要加上这些判断，很浪费时间，倒不如把这些跳转整合到一个路由中，将当前的absoluteString输入进去，怎么去跳转由路由去决定。

NSURL的几个属性

好像有点跑题，但是我们还是先讲一下NSURL的几个属性

  • scheme 协议
  • host 域名
  • absoluteString 完整的url字符串
  • relativePath 相对路径
  • port 端口
  • path 路径
  • pathComponents search
  • query 参数

结构图

xxx.plist

该plist文件负责将APP中需要路由跳转的一组{URL Scheme,目的地的VCClassName} 先给保存起来，可以有多个文件plist文件，可以埋点注册。

注册.plist文件

在APP启动的时候将提前写好的plist通过GCNavigtor(单例)注册到GCRouterManager中，每一组{URL Scheme,目的地的VCClassName} 注册成一个GCRouter对象负责在触发CGNavigator的push方法时提供目的VC的className

在GCRouterManager中，有一个NSMutableDictionary去保存所有的GCRouter对象，用url.path作为该字典的key去保存具有相同的Path的GCRouter数组。

同理注销{URL Scheme,目的地的VCClassName} 时也是利用这个字典去判断，将GCRouter移除。

由于这个字典是{NSString:NSArray}格式，有可能在注册和注销的线程，操作同一块数据资源，此时可能会出现一些并不存在的数据或者错误，所有在write这块数据资源的时候需要加锁

利用信号量的 wait&signal去限制当前最多只有一条线程调用到这块数据

跳转

根据上面的描述，业务层是用plist，通过GCNavigator将

{URL Scheme,目的地的VCClassName}

注册到GCRouterManager中。

这样我们可以实现只有GCNavigator暴露在业务层中，不需要知道GCRouterManager中的内容，我们只需要调用GCRouterManager的push和present方法将WKWebView回调的URL，和目的tab传入，就可以实现跳转功能

在GCNavigator中利用

GCRouter *router = [GCRouterManager fetchRouterForURL:[url ignoreHTTPS]];

通过这个方法去获取到具体的GCRouter中的目的VCClass

在这方法中因为有可能存在URL path相同但是指向不同的VCClass的GCRouter，我们之前是把他们放在同一个NSArray中，这个时候我们可以将for循环一次通过URL去和数组中的GCRouter的URL数据做比较，判断去获取具体的GCRouter

跳转push是利用选中的tab中的navigationC去跳转，将其加在最上面

模态present是利用选中的tab中的navigationC最上面的VC，present出来

回调

跳转完成，需要将URL中的数据回调给目的VC去发送网络请求

所以需要在目的VC中遵循GCNavigatorDelegate代理,并实现代理方法

利用url的query属性获取请求的参数，并转换成参数字典
